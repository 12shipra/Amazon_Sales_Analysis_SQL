Amazon Sales Analysis (SQL Project)
Description
Developed a comprehensive sales analysis project using MySQL to analyze Amazon’s retail sales data. The project involved creating and managing a relational database to store detailed sales information, including invoice details, product lines, customer demographics, sales transactions, and branch-level data. Using SQL, various business questions were answered to uncover sales trends, customer behavior, and revenue drivers.

Key Responsibilities
Database Schema Design: Designed and implemented a sales table schema to efficiently store and manage Amazon’s sales data.
SQL Data Analysis: Wrote advanced SQL queries to extract, aggregate, and analyze sales data, providing insights on product performance, customer segments, and sales trends.
Feature Engineering: Created new features such as time of day, month name, and day name from timestamps to facilitate temporal analysis.
Business Problem Solving: Addressed key business questions such as identifying top-selling products, peak sales months, and payment method popularity.
Branch and City Performance Evaluation: Analyzed branch-wise and city-wise performance to identify high-revenue locations and optimize sales strategies.
Customer Segmentation: Segmented customers by type and gender to analyze purchasing patterns and revenue contributions.
Key Skills and Tools
MySQL: Extensive use of SQL commands such as SELECT, GROUP BY, JOIN, CASE, and window functions like RANK() for data manipulation and analysis.
Database Management: Experience in schema creation, table design, and data updating through ALTER and UPDATE commands.
Data Aggregation and Analysis: Aggregated transactional data to generate actionable insights on revenue, product demand, and customer behavior.
Temporal Analysis: Used date and time functions to analyze sales patterns across months, days, and times of day.
Conditional Logic: Applied CASE statements for classification and segmentation tasks.
Achievements
Identified key sales drivers, including best-performing products and peak sales periods, enabling data-driven decision making.
Improved database efficiency and query performance through thoughtful schema design and indexing.
Enhanced understanding of customer behavior patterns by analyzing segmentation by gender, customer type, and time of purchase.
Project Impact
This project provided actionable insights into Amazon’s sales trends, customer segmentation, and branch performance. These insights can help optimize marketing strategies, improve inventory management, and increase overall revenue through targeted business decisions. The project also strengthened practical SQL skills and demonstrated the application of feature engineering in a business context.
